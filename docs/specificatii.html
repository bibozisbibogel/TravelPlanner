1. Prezentare generală

1.1 Descrierea aplicației

TravelPlanner este o aplicație web de planificare a călătoriilor care permite utilizatorilor să creeze, gestioneze și urmărească itinerarii complete pentru destinații globale. Aplicația oferă funcționalități de căutare a zborurilor, cazărilor și activităților, recomandări inteligente de restaurante și experiențe culinare locale, urmărire a bugetului pe categorii, vizualizare interactivă a itinerariului și gestionarea profilului utilizatorului.

Inspirată din proiectul Travel Companion, aplicația este reconstruită pe o stivă tehnologică accesibilă: frontend HTML/CSS/JavaScript cu Bootstrap 5, backend ASP.NET Core și bază de date PostgreSQL hospedată pe Supabase, renunțând la dependențele de framework-uri JavaScript complexe în favoarea compatibilității și simplității.

1.2 Obiective principale
  • Planificarea completă a unei călătorii (destinație, date, buget, activități, cazare)
  • Vizualizarea itinerariului zi cu zi cu timeline interactiv
  • Descoperirea de restaurante și specialități culinare locale pentru fiecare destinație
  • Urmărirea în timp real a bugetului pe categorii de cheltuieli
  • Explorarea și descoperirea de noi destinații cu filtrare avansată
  • Gestionarea completă a profilului utilizatorului și preferințelor
1.3 Utilizatori țintă
  • Turiști individuali – călătorii solo sau în cuplu
  • Familii – planificarea vacanțelor cu copii
  • Groupuri de prieteni – organizarea călătoriilor de grup
  • Călători de business – gestionarea deplasărilor profesionale

3. Arhitectura aplicației

3.1 Pattern arhitectural

Aplicația urmează pattern-ul MVC (Model-View-Controller) pe backend, cu o separare clară între:

  • Modele – entitățile de date (User, Trip, Itinerary, Budget)
  • View-uri – paginile HTML statice servite din frontend
  • Controller-e – endpoint-urile REST din ASP.NET Core Web API

3.3 Comunicarea frontend–backend

Frontend-ul comunică cu backend-ul exclusiv prin cereri HTTP asincrone (fetch API) cu format JSON. Autentificarea se realizează prin JWT token stocat în localStorage, trimis în header-ul Authorization: Bearer <token> la fiecare cerere protejată.

Fiecare dintre cele 7 pagini de conținut conține obligatoriu:

8. Autentificare & Securitate

Autentificare JWT
  • Token JWT generat la login cu expirare de 24 ore
  • Stocat în localStorage pe client
  • Trimis în header Authorization: Bearer la fiecare request
  • Validat de middleware ASP.NET Core pe rutele protejate
  • Refresh token opțional pentru sesiuni prelungite
Securitate parolă
  • Hashing cu BCrypt (cost factor 12)
  • Lungime minimă: 8 caractere
  • Cel puțin o literă mare, o cifră
  • Fără stocare în plain text
  • Rate limiting pe endpoint-ul de login
Măsuri de securitate generale
CORS

Politică CORS configurată pentru a permite doar originile autorizate (frontend).

Input Validation

Validare server-side cu Data Annotations și FluentValidation. Protecție împotriva SQL Injection prin EF Core parametrizat.

HTTPS

Redirect automat HTTP → HTTPS în producție. Certificat SSL/TLS obligatoriu.
